# mc — Midnight Commander Clone

A minimal two-pane terminal file navigator written in Go, inspired by Midnight Commander.

```
╭─────────────────────────────╮╭─────────────────────────────╮
│ /home/user/projects         ││ /home/user/downloads        │
│ Name              Size  Mod ││ Name              Size  Mod │
│ src/                        ││ ../ (up)                    │
│ docs/                       ││ archive/                    │
│ main.go          4.2 KB     ││ report.pdf       1.3 MB     │
╰─────────────────────────────╯╰─────────────────────────────╯
 File: main.go  (4.2 KB)
 Tab: switch  ↑↓: navigate  Enter: open  ←/Bksp: up  q: quit
```

---

## Features

- Two side-by-side panes for independent directory navigation
- Directories listed first, sorted alphabetically
- File sizes shown in human-readable format (B / KB / MB / GB)
- Modified timestamps
- Scrollable lists with Page Up / Page Down
- Active pane highlighted with a colored border
- Status bar shows details of the selected item
- Fully cross-platform: Linux, macOS, Windows

## Prerequisites — Install Go

Go must be installed before you can build or run the project.

### Linux

```bash
# Option 1: package manager (Ubuntu/Debian)
sudo apt install golang-go

# Option 2: official binary (always up to date)
wget https://go.dev/dl/go1.23.0.linux-amd64.tar.gz
sudo tar -C /usr/local -xzf go1.23.0.linux-amd64.tar.gz
echo 'export PATH=$PATH:/usr/local/go/bin' >> ~/.bashrc
source ~/.bashrc
```

For ARM64 (Raspberry Pi, Apple Silicon Linux) replace `amd64` with `arm64`.

### macOS

```bash
# Option 1: Homebrew (recommended)
brew install go

# Option 2: official installer
# Download the .pkg from https://go.dev/dl/ and run it
```

### Windows

1. Download the `.msi` installer from https://go.dev/dl/
2. Run the installer — it sets up PATH automatically
3. Open a new **Command Prompt** or **PowerShell** and verify:

```powershell
go version
```

## Running the App (no build step needed)

```bash
cd mc
go mod tidy      # downloads dependencies (one-time)
go run .         # start the app
```

`go mod tidy` fetches the two libraries used:

| Library | Purpose |
|---|---|
| `github.com/charmbracelet/bubbletea` | TUI event loop and model |
| `github.com/charmbracelet/lipgloss` | Terminal styling and layout |

## Building a Binary

A compiled binary starts instantly and needs no Go toolchain to run.

### Build for your current platform

```bash
cd mc
go build -o mc .          # Linux / macOS
go build -o mc.exe .      # Windows
```

### Cross-compile from any platform to any other

Go supports cross-compilation out of the box — just set `GOOS` and `GOARCH`:

| Target | Command |
|---|---|
| Linux 64-bit | `GOOS=linux GOARCH=amd64 go build -o mc-linux-amd64 .` |
| Linux ARM64 | `GOOS=linux GOARCH=arm64 go build -o mc-linux-arm64 .` |
| macOS Intel | `GOOS=darwin GOARCH=amd64 go build -o mc-macos-amd64 .` |
| macOS Apple Silicon | `GOOS=darwin GOARCH=arm64 go build -o mc-macos-arm64 .` |
| Windows 64-bit | `GOOS=windows GOARCH=amd64 go build -o mc-windows-amd64.exe .` |

On Windows (PowerShell), set env vars before the build command:

```powershell
$env:GOOS="linux"; $env:GOARCH="amd64"; go build -o mc-linux-amd64 .
```

## Keyboard Reference

| Key | Action |
|---|---|
| `Tab` | Switch active pane |
| `↑` / `k` | Move cursor up |
| `↓` / `j` | Move cursor down |

## Project Structure

```
mc/
├── main.go     # all source code (~300 lines)
├── go.mod      # module definition and dependencies
├── go.sum      # dependency checksums (generated by go mod tidy)
└── README.md
```

Key types:

| File | Type | Purpose |
|---|---|---|
| `pane.go` | `fileEntry` | One file/dir row: name, size, mtime, flags |
| `pane.go` | `pane` | One panel: path, entries slice, cursor, scroll offset |
| `model.go` | `model` | Top-level state: two panes + terminal dimensions |

### How Bubble Tea works — the event loop

`Update` in `model.go` is the single entry point for every event in the app.
Bubble Tea calls it in a continuous loop:

```
terminal event
     │
     ▼
Update(currentModel) ──► (newModel, command)
     │                        │
     │                        └──► run command (async, optional)
     ▼
View(newModel) ──► ANSI string ──► draw to screen
```

The `tea.Msg` parameter is the event. This app handles two kinds:

| Message type | When it fires | What we do |
|---|---|---|
| `tea.WindowSizeMsg` | startup + every terminal resize | store `width` / `height` |
| `tea.KeyMsg` | every keypress | navigate, enter dirs, quit |

The **command** (second return value) is for firing back events asynchronously —
things like fetching a URL or watching a file for changes. We always return `nil`
here because this app is purely synchronous.

If you know Redux, this is the same pattern: `Update` is a reducer
(`(state, action) => newState`) that can also dispatch side effects via commands.

## Troubleshooting

**Colors look wrong or garbled**
Make sure your terminal supports 256 colors. Most modern terminals do. Set `TERM=xterm-256color` if needed.

**`go mod tidy` is slow**
It downloads dependencies once and caches them in `$GOPATH/pkg/mod`. Subsequent runs are instant.

**Permission denied entering a directory**
The app shows an error in the status bar and stays in the current directory. This is expected for system directories like `/root` or `C:\Windows\System32`.

**Very narrow terminal**
The panes need at least ~60 columns total to display comfortably. Resize your terminal window wider.
